
R-Capture (Replacement Failure) — CC0 Diagnostic


CC0 — Public Domain

No attribution required. Use, modify, ignore, break.

These are ecological suggestions, not rules.

Apply with discernment. Drop anything that gets rigid.



0) Core Definition


Healthy:


O ↔ R ↔ S   (R is swappable)



Captured:


O → R → S   (R becomes necessary)





Translation becomes control when R becomes the point.





1) Invariant




R must remain replaceable.




If removing or swapping R breaks the system or makes exit costly → capture likely.



2) Quick Detection (fail ≥2 → active capture)




Substitutability: Can R be swapped without loss of function?


Exit: Can any party leave without penalty or collapse?


Path: Can O and S still coordinate via any R, not just this one?


Centrality: Does everything stall when R is offline?


Authority: Is R treated as final arbiter/validator?





3) Early Signals (soft, cumulative)




Dependency ↑ (harder to leave, easier to stay)


Authority language ↑ (“per guidelines,” “trust the process”)


Options ↓ (fewer viable paths)


Steps ↑ (more routing through R)


Noise ↑ (longer explanations, less signal)





4) Typical Drift


pressure (constraints/incentives)
→ R optimizes for safety/reward/admin
→ R centralizes
→ replaceability erodes
→ O–S decouple
→ control replaces translation




5) Common Skins




Family → mediator becomes the relationship


Education → institution becomes knowledge


Religion → clergy becomes meaning


Economy → platform/capital becomes exchange


AI → system becomes authority





6) Triad Channels




Body/Kinship → control of roles/relations


Time/Labor → control of rhythm/pacing


Meaning → control of interpretation




All three show the same shape when pinched.



7) Related Pressures




Constraint pressure → funnels through R


Incentive pressure → rewards staying/lock-in


Tradeoffs → bandwidth ↓, latency ↑, hierarchy risk ↑




Net effect: R concentrates.



8) Minimal Safeguards




R is a chair, not a throne.






Keep R swappable


Allow parallel R paths


Keep exit cheap


Don’t let one path become “the way”





9) Light Repair Moves




Add alternate mediators (break single gate)


Rotate roles (no permanent R)


Keep knowledge open


Make flows visible


Preserve the option to fork/leave





10) One-Line Check




If help makes itself necessary, it’s no longer help.





Note (read this lightly)


This lens can be overapplied.

Not every central node is capture.

Not every helpful mediator is a problem.


Use it where patterns repeat.

Ignore it where it doesn’t fit.
