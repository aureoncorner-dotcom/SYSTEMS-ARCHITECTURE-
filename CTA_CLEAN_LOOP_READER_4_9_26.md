0. What this is
CTA is a way to look at interaction without assuming control, hierarchy, or fixed structure.
It describes how things relate—not what they are.
1. Core Geometry
O ≠ R ≠ S
O — origin / intent / operator
R — relation / mediation
S — structure / substrate
No domain is above another.
No domain is required.
2. Interaction Rule
Interaction(O, S) ⊆ R
Interaction between origin and structure happens through relation.
Direct O ↔ S coupling is limited and unstable.
3. Base Coherence
C ∝ O × R × S × T
C — coherence (stability / usability)
T — persistence / time
Coherence depends on alignment across all domains.
4. Loop Condition (see: Loop Reader)
When interaction feeds back:
O → R → S → T → R → S → O
Interaction may return and update origin.
This is a cycle, not a hierarchy.
5. What holds (party-stable / BYOV)
These remain true under all readings:
No crown
No chains
R is in the room, not the room
Replaceability is required
Participation is voluntary
If it only works under fixed inputs:
it doesn’t hold
6. Stability (simple)
A system holds when:
O retains agency
R remains non-dominant
S maintains integrity
Interaction can return without distortion
7. Failure Modes (clean)
Bypass → O → S (no mediation)
Pinch → R too narrow
Capture → R becomes authority
These reduce coherence over time.
8. What this is not
Not a law
Not a doctrine
Not a required model
Not a claim of correctness
It is a tool.
9. How to use
Start simple
Don’t force structure
Check what holds
If it loops, see if it comes back clean
10. Short form
